###现状
- 手工代码效率低
- 质量差，有时是为了单测而单测，没有达到单测的初衷


###目标
####通过一切手段达到单元测试代码的自动化
#####方向1：通过约定、注解等方式约束源代码，
#####方向2：机器学习，通过学习已有的单测代码，写新的代码

###用DDD的方式开发
####领域建模
- 用例
    - 不用的单元测试工具产生的代码不同，初期考虑Junit即可。
    - 代码生成
        - 全覆盖模式，生成新的
        - 选择性生成
